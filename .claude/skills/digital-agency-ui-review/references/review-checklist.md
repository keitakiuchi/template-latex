# レビュー観点チェックリスト

このチェックリストは、デジタル庁の関連資料をもとにした実務用の要約。正式な適合確認ではなく、設計・レビューの観点整理として使う。

## 1. 利用者視点・サービス設計

- 主な利用者が明確か。例: 住民、事業者、職員、支援者、初回利用者、外国人、障害のある利用者。
- 利用者が達成したいタスクが明確か。
- 入口、途中離脱、再開、完了後の次の行動が分かるか。
- 手続きの期限、必要書類、対象者、問い合わせ先が分かるか。
- 利用者が間違えやすい箇所、取り返しのつきにくい箇所を先回りしているか。
- 利用者からのフィードバックを受け、改善する運用があるか。

## 2. 情報設計・コンテンツ

- ページタイトルと見出しだけでページの目的が分かるか。
- 重要情報がページ上部にあるか。
- 専門用語や制度用語に、やさしい説明があるか。
- リンク文言は「こちら」だけでなく、遷移先や行動が分かるか。
- 更新日、情報源、担当、問い合わせ先、関連手続きが分かるか。
- 公開後の更新、廃止、アーカイブ、記録管理の方針があるか。
- 生成AIを使ったコンテンツは、人間による確認と責任範囲が明確か。

## 3. ナビゲーション・画面構造

- 現在位置、戻る/進む、手順、完了までの残りが分かるか。
- パンくず、見出し、ランドマーク、ページ内ナビゲーションが適切か。
- モバイル幅でも読む順序、操作順、情報量が破綻しないか。
- ボタンとリンクの役割が混ざっていないか。
- 主要アクションと副次アクションの見た目・位置・文言が分かりやすいか。

## 4. 入力フォーム

- 各入力欄に可視ラベルがあるか。
- 必須/任意が分かるか。
- 入力条件、桁数、文字種、形式、記入例が入力前に分かるか。
- 記入例や重要な説明をプレースホルダーだけにしていないか。
- 文字数制限が入力前に分かり、必要に応じて入力中もカウンターがあるか。
- `autocomplete`、`inputmode`、適切なキーボード種別など、入力支援を使っているか。
- エラー時に、どの欄がなぜエラーか、どう直せばよいか分かるか。
- 送信後も入力済みの内容が保持されるか。
- エラー一覧から該当欄へ移動できるか。
- 確認画面や送信前の修正導線が分かるか。

## 5. アクセシビリティ

- HTMLの意味構造が適切か。見出し、リスト、表、ボタン、リンク、フォーム部品を正しく使っているか。
- キーボードだけで全ての機能を操作できるか。
- フォーカス位置が見えるか。
- スクリーンリーダーで読み上げ順が意味の通る順序になっているか。
- 画像、アイコン、グラフに目的に応じた代替テキストや説明があるか。
- 色だけで状態や意味を伝えていないか。
- 文字色と背景色のコントラストが不足していないか。
- 文字拡大、ズーム、スマートフォン表示で情報や操作が失われないか。
- 動き、点滅、自動更新が利用者の負担になっていないか。
- エラー、成功、読み込み中、保存済みなどの状態が支援技術にも伝わるか。

## 6. デザインシステム・UIコンポーネント

- デジタル庁デザインシステムの基本デザインやコンポーネントで解決できるか。
- 色、文字、余白、アイコン、ボタン、フォーム、ナビゲーションが一貫しているか。
- 独自コンポーネントを作る場合、既存コンポーネントとの差分と理由が明確か。
- 状態表現があるか。例: 通常、ホバー、フォーカス、押下中、無効、エラー、完了。
- 視覚的な見た目だけでなく、支援技術に伝わる名前・役割・状態があるか。

## 7. ダッシュボード・データ可視化

- 誰が何を判断し、どの行動につなげるためのダッシュボードか明確か。
- 主要指標、補助指標、詳細データの優先順位があるか。
- グラフの種類が目的に合っているか。比較、推移、構成比、分布、地理情報を混同していないか。
- タイトル、単位、期間、データ出所、更新日、注釈が分かるか。
- 軸の開始位置、目盛、並び順、集計単位が誤解を生まないか。
- 色だけで増減や分類を伝えていないか。
- 凡例、直接ラベル、注釈が読みやすいか。
- 関係者合意のために、要件定義、プロトタイプ、チェックリストを使っているか。

## 8. 実装・テスト

- HTML/CSS/JavaScriptが、意味構造を壊さずに実装されているか。
- 自動テストだけでなく、キーボード操作、スクリーンリーダー、モバイル、ズーム、実データで確認しているか。
- エラー系、空データ、長い文言、多言語、低速回線、セッション切れを確認しているか。
- ログ、問い合わせ、利用者フィードバックを改善に使えるか。

## 重大度の判定目安

重大:
- 利用者が手続き・申請・閲覧を完了できない。
- 障害のある利用者やスマートフォン利用者が主要機能を使えない。
- 誤入力、誤解、個人情報漏えい、取り返しのつかない操作につながる。

中:
- 操作はできるが迷いやすい、再入力が多い、説明が不足している。
- エラーから回復しにくい。
- 情報はあるが見つけにくい。

軽微:
- 表記ゆれ、余白、視線誘導、ラベル統一、補足説明など。
