# 公式資料ソースマップ

確認日: 2026-05-13

このファイルは、レビューや設計支援時に参照するデジタル庁関連資料の対応表。最新性が必要な場合は、必ず公式ページを再確認する。

## 中核資料

### デジタル庁デザインシステムβ版

URL: https://design.digital.go.jp/dads/

使う場面:
- Webサイト、Webアプリケーション、オンラインサービスの画面設計。
- カラー、タイポグラフィ、余白、UIコンポーネント、作例、アクセシビリティ観点の確認。
- Figmaデータ、コードスニペット、イラスト・アイコン素材の利用方針確認。

注意:
- デザインシステムを使えば全てのアクセシビリティが自動的に満たされるわけではない。
- サービス固有のブランド、情報構造、コンテンツ、運用ルールは別途整理する。

### 行政サービスにおける利用者視点導入ガイドブック

URL: https://www.digital.go.jp/resources/user-centered-approach-guidebook/practices

使う場面:
- サービス全体を利用者視点で設計する。
- 利用者調査、利用品質目標、調達、改善サイクルを考える。
- どのガイドライン/ガイドブックを読むべきか整理する。

## 標準ガイドライン・ガイドブック

標準ガイドライン一覧:
https://www.digital.go.jp/resources/standard_guidelines

### DS-670.1 ユーザビリティガイドライン

位置づけ: Normative（順守対象の規範的文書）
最終改定日: 2025-09-30

使う場面:
- 政府情報システムのUI要件、レビュー、受け入れ基準を作る。
- 誤操作を起こしにくく、安全で、利用ニーズに合うUIを設計する。
- 入力フォーム、バリデーション、入力支援の確認。

関連:
- DS-670.1遵守状況チェックリスト
- UIチェックリスト

### DS-671.1 ユーザビリティ導入ガイドブック

位置づけ: Informative（参考文書）
策定日/最終改定日: 2025-10-16

使う場面:
- 利用者中心のUI設計を初めて整理する。
- 基本原則、手法、レビュー観点を説明する。
- 発注者、受託者、レビュー担当者向けの共通理解を作る。

### DS-671.2 ウェブアクセシビリティ導入ガイドブック

URL: https://www.digital.go.jp/resources/introduction-to-web-accessibility-guidebook
位置づけ: Informative（参考文書）
策定日: 2025-10-16

使う場面:
- Webアクセシビリティの基本を説明する。
- 行政官、事業者、非技術者向けに確認ポイントを整理する。
- JIS X 8341-3:2016、スマートフォン対応、支援技術との関係を扱う。

### DS-672.1 ウェブアクセシビリティ広報向けガイドブック

位置づけ: Informative（参考文書）
策定日/最終改定日: 2025-10-16

使う場面:
- Webサイト構築後に追加されるコンテンツのアクセシビリティを確認する。
- SNS、動画、画像、広報資料、投稿文、代替テキストを確認する。
- 広報向けコンテンツチェックリストを使う。

### DS-680.1 ウェブサイトガイドライン

位置づけ: Normative（順守対象の規範的文書）
最終改定日: 2025-09-30

使う場面:
- 各府省などが設置するWebサイトの構築、調達、品質管理。
- プライバシー、セキュリティ、アクセシビリティ、利用者中心の品質確認。

### DS-680.2 ウェブコンテンツガイドライン

位置づけ: Normative（順守対象の規範的文書）
最終改定日: 2025-09-30

使う場面:
- Webで発信する情報の種類、文章表現、情報アクセシビリティ、オープンデータ、生成AI利用、コンテンツのライフサイクル管理、記録管理。
- 公開ページ、制度説明、FAQ、資料掲載、更新/廃止の運用設計。
