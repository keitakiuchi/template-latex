# Writing for Top Journals: 実証系心理学・社会科学のための統合ライティングガイド

作成日: 2026-04-01  
対象: 心理学、応用心理学、産業保健、組織行動、マネジメント、社会科学の実証研究  
前提: とくに *Psychological Science*、*Journal of Applied Psychology*、*Academy of Management Journal* を念頭に置いているが、多くの原則は他の査読誌にも応用できる  
統合元: `writing_for_top_journals_guide.md`、`manuscript_and_sap_guidance.md`、`general_statistical_analysis_plan_guide.md` のうち、論文執筆・原稿準備・分析計画との接続に関わる内容

---

## はじめに

トップジャーナル論文は、単に「有意差が出た論文」ではありません。むしろ、

1. **重要な問い**を扱っていること
2. **理論的貢献**が明確であること
3. **研究デザインが問いに適合**していること
4. **方法・分析・記述が十分に透明**であること
5. **専門外の読者にも価値が伝わる**こと

が強く求められます。

この統合版では、単なる文章術ではなく、

- 何を主張する論文なのかを最初に固定すること
- 原稿と analysis plan / SAP を整合させること
- 研究デザインに応じて「言ってよいこと」と「言いすぎになること」を区別すること
- 査読で問われやすい点を先回りして潰すこと

まで含めて、論文執筆の実務を一つの流れとして整理します。

## 0. まず最初に固定すべきこと

原稿も解析計画も、最初に固定すべきなのは「使う手法」ではなく、**研究の問いの型**です。ここが曖昧だと、方法、結果の見せ方、考察の結論、査読対応のすべてがぶれます。

### 0.1 研究の問いの型

最低限、次のどれが主目的かを明示します。

- **記述（descriptive）**
  - 何がどれくらい起きているか、分布や実態を示したい研究
- **関連・説明（associational / explanatory）**
  - 変数間の関連を調べたい研究
- **因果効果推定（causal effect estimation）**
  - ある曝露や介入が結果にどう影響したかを推定したい研究
- **予測・分類（prediction / classification）**
  - 将来のアウトカムやクラスを当てたい研究
- **方法論・シミュレーション（methodological / simulation）**
  - 解析法の性質や条件ごとの性能を調べたい研究
- **エビデンス統合（evidence synthesis）**
  - 既存研究を体系的に集め、比較・統合したい研究

同じデータ、同じ回帰分析でも、問いの型が違えば、良い原稿の条件も、良い analysis plan の条件も、考察で言える範囲も変わります。

### 0.2 書き始める前に一文ずつ言えるようにすること

原稿を書く前に、少なくとも次の4点を一文ずつで言えるようにしておくと、全体がぶれにくくなります。

- **研究課題**
  - この論文は何の問題に答えるのか
- **既存研究の不足**
  - 何が分かっていて、何がまだ分かっていないのか
- **本研究の貢献**
  - その不足を理論、方法、データのどこで埋めるのか
- **主張の射程**
  - どこまで言えて、どこからは言えないのか

この4点が曖昧なまま書き始めると、導入は長く、理論部は説明的になり、Methods は羅列的になり、Discussion は過大主張になりやすいです。

### 0.3 投稿先と報告ガイドラインを先に決める

原稿は最後に雑誌へ合わせるより、**最初から投稿先の期待に沿って設計する**方が強いです。研究タイプごとの代表的なガイドラインは次の通りです。

- 観察研究: STROBE
- routine data を使う観察研究: RECORD / RECORD-PE
- 予測モデル研究: TRIPOD / TRIPOD+AI
- 系統的レビュー / メタアナリシス: PRISMA 2020
- 系統的レビューのプロトコル: PRISMA-P
- 観察研究のメタアナリシス: MOOSE
- ランダム化比較試験: CONSORT、SPIRIT
- 質的研究・混合研究法: APA JARS、SRQR、COREQ
- 統計記述全般: SAMPL

大事なのは、ガイドラインを投稿直前の形式チェックにだけ使わないことです。構成案の時点でチェックリストを開き、どの情報を本文に置き、どの情報を補足資料に回すかを決めておくと、査読での取りこぼしがかなり減ります。

### 0.4 analysis plan / SAP との関係

良い原稿は、良い analysis plan / SAP を背後に持っています。SAP は臨床試験専用文書ではなく、より広い意味での **analysis plan** として、

- 主解析を先に固定する
- 重要な分岐条件を先に決める
- 確証的解析と探索的解析を区別する
- 結果を見てから都合よく解析を選ぶ余地を減らす

ための文書です。

トップジャーナル向け原稿では、Methods の説得力は、「何をしたか」の記述量よりも、**どの自由度を事前に潰していたか**で決まることが少なくありません。

## 1. トップジャーナルが実際に見ているもの

### 1.1 「面白い結果」より「重要な問い」

トップジャーナルは、驚く結果そのものよりも、**その研究がどの問いに答えているのか**を見ます。掲載基準として、広い理論的・実証的意義、方法論的厳密性、一般読者への relevance が明示されていることが多く、評価の中心は「有意だったか」ではなく、「問いと設計が重要か」です。

### 1.2 貢献は「結果」ではなく「理解の更新」

問われるのは、「何が見つかったか」だけではありません。**その結果によって既存理解がどう変わるのか**が必要です。新規性だけでなく、学術的・実践的に何を可能にするのかまで示さないと、トップ誌では弱く見えやすいです。

### 1.3 狭い専門内でしか通じない書き方は不利

トップ誌ほど、専門性と同時に**広い可読性**を求めます。専門家だけが分かる略語や概念の省略は、査読者の好意を前提にした書き方になりがちです。専門性は下げずに、読み手の認知負荷は下げる必要があります。

### 1.4 近年は透明性が「加点」ではなく「前提」

心理学・社会科学では、透明性はもはや「あれば望ましい」ではなく、**査読可能性そのものの一部**です。研究透明性に関する独立セクション、データ・マテリアル・分析スクリプトの共有、AI 利用の開示などは、追加の美点ではなく、信頼性の基盤として見られます。

## 2. 論文全体の基本設計

### 2.1 論文は一本の論理としてつなぐ

トップジャーナル論文は、細部がよく書けているだけでは足りません。基本的には次の流れを、一つの論理線でつなげる必要があります。

1. 重要な現象・問題を提示する
2. 既存研究が何を明らかにしているかを整理する
3. 不足・矛盾・未解決点を特定する
4. 本研究がどう答えるかを明示する
5. 適切な方法で検証する
6. 結果を簡潔に示す
7. 理論的・実践的含意を、過不足なく論じる

### 2.2 主解析は原則として一つに近づける

主研究課題が複数あり、主解析も複数あり、考察ではそのうち都合のよい結果だけを強調する、という形は非常に弱いです。

- 中心的な問いは一つか、ごく少数に絞る
- 主解析を一つに固定する
- 他は副次解析、探索解析、感度分析として位置づける

この整理ができていないと、査読で「結局この論文の本線は何か」が問われます。

### 2.3 理論的貢献は早めに明示する

導入の後半、遅くとも理論節の終盤までには、

- 何が足りないのか
- なぜその不足が重要なのか
- 本研究はその理解をどう更新するのか

をかなり明確に書いた方がよいです。貢献を最後まで引っ張ると、「よくある知見の追加報告」に見えやすくなります。

### 2.4 方法は「再現できるか」と「主張に見合っているか」の両方を満たす

Methods は単に手順を書くだけの場所ではありません。**研究課題に対して、その方法が十分だったことを読者が判断できるようにする場所**です。

### 2.5 書けることより、言ってよいことを守る

記述研究、関連研究、因果推定研究、予測研究、シミュレーション研究、メタアナリシスでは、Discussion で言える範囲が違います。トップ誌では、「過大主張がない」こと自体が知的成熟の一部として見られます。

## 3. セクション別の書き方

### 3.1 タイトル

タイトルは、派手さよりも**何を調べた論文かが一読で分かること**が重要です。

避けたい方向:

- 曖昧なキャッチフレーズ
- データが支えていない広い主張
- デザインや文脈が見えない題名

望ましい方向:

- 何について
- どの関係を
- どの文脈で
- どのような設計で

を読み手が把握できる題名

### 3.2 アブストラクト

良いアブストラクトは、次の要素を短く押さえます。

- 背景となる問い
- 研究目的
- 方法の骨格
- 主結果
- 理論的含意
- 必要なら主要な限界

要旨で気を付ける点:

- タイトルと要旨で研究の型を誤魔化さない
- 観察研究で因果的に言い切りすぎない
- 有意差だけでなく、効果量と不確かさを書く
- 事前指定した解析と探索的解析を混ぜない
- 本文で言えないことを要旨で先走って言わない

### 3.3 Introduction

導入は、単なる背景説明ではありません。読者に「この問いは読む価値がある」と思わせる場所です。基本形は次の4段階で考えると整理しやすいです。

1. 現象の重要性
2. 既存研究の到達点
3. 不足・矛盾・未解決点
4. 本研究の応答

導入で明確にしたいこと:

- 本研究の研究課題
- 何がまだ十分に説明されていないか
- 理論的貢献
- 方法上の強み
- 実践的意義
- 主張の射程

導入でよくある失敗:

- 背景説明が長く、研究課題がなかなか出てこない
- 先行研究を著者紹介のように並べる
- ギャップは言うが、なぜ重要かを説明しない
- 目的が多すぎて焦点がぼける
- 強いことを言いすぎて後半で回収できない

### 3.4 理論部・先行研究レビュー・仮説構築

トップジャーナルでは、この部分は「文献レビュー」ではなく、**理論的に駆動された議論**であることが求められます。大事なのは、何が知られているかの要約ではなく、**なぜ本研究の主張が理にかなうかの説明**です。

良い理論節の特徴:

- 段落ごとに一つの論点がある
- 各段落の冒頭で主張が明示される
- 引用はその主張を支えるために使われる
- 研究間の一致、不一致、境界条件が整理される
- 最後に本研究の予測や問いへ橋渡しする

避けたい理論節:

- 研究Aはこう、研究Bはこう、研究Cはこう、で終わる
- X と Y を別々に長く説明するが、なぜ X が Y を生むかが弱い
- 多数の理論を並べるが、どれが主理論か分からない
- 関連しそうだから仮説を立てたように見える
- 先行研究の羅列で、自分の論理が見えない

### 3.5 Methods

Methods の役割は、「何をしたか」だけでなく、**研究課題に対して、その方法が妥当だったことを示すこと**です。最低限、次を明確にします。

#### 3.5.1 対象・単位・選定

- 誰、あるいは何を分析単位にしたのか
- どこからサンプルを得たのか
- 組み入れ基準と除外基準は何か
- いつからいつまでのデータを使ったのか
- 最終解析対象は元の対象集団とどう違うか

メタアナリシスなら通常は「研究」や「効果量」、シミュレーション研究なら「反復」や「シナリオ」が分析単位です。ここを曖昧にしないことが重要です。

#### 3.5.2 変数・測定・定義

- 主要アウトカム
- 副次アウトカム
- 主要曝露、介入、予測因子、説明変数
- 共変量、交絡候補
- 派生変数の作り方
- 測定のタイミング

routine data や既存データ研究では、対象者抽出に使ったコードやアルゴリズム、曝露・アウトカム・交絡因子の定義に使ったコードやアルゴリズムを、可能な範囲で一覧化すると強くなります。

#### 3.5.3 時間構造

観察研究や二次データ解析では、時間の向きが曖昧だと急に弱く見えます。

- いつを起点にしたのか
- 曝露はいつ測ったか
- アウトカムはいつから追ったか
- フォローアップ終了条件は何か
- ラグ期間や洗い出し期間を置いたか

#### 3.5.4 データ源と前処理

特に既存データ研究では、データ源の説明は Methods の中核です。

- データベース名、版、抽出日
- どの機関・システムから得たか
- 変数の可用性と限界
- データクリーニングの手順
- データ連結があるなら、その方法と質評価
- 欠測、重複、矛盾値への対応

#### 3.5.5 倫理・透明性

- 倫理審査の有無と承認情報
- 同意取得の扱い
- 事前登録の有無
- プロトコルや SAP の有無
- データ・コード・資料の共有方針
- 資金源、利益相反
- 生成 AI を使ったなら、その用途、範囲、責任の所在

#### 3.5.6 統計・分析の書き方

多くの原稿で弱くなりやすいのは、統計欄が分析法の名前の列挙になっていることです。査読者が見たいのは、**結果を見てから都合よく選べた余地がどれくらい残っているか**です。

最低限、次を明示したいです。

- 主解析は何か
- それに対応する推定量は何か
- どの効果指標を使うか
- どの共変量を、なぜ入れるか
- 欠測値をどう扱うか
- モデルの前提をどう確認するか
- 前提が崩れたら何をするか
- サブグループ解析をどこまで事前指定したか
- 感度分析を何本入れるか
- 使用ソフトウェアとバージョン

### 3.6 Results

結果節では、分析を「やった順」に並べるのではなく、**研究課題に答える順**に並べた方が読みやすくなります。

意識したいこと:

- まず、何が何件あったかの流れを示す
- 次に、分析対象の特徴を示す
- その後、主解析、副次解析、探索解析の順に出す
- 欠測、除外、脱落、逸脱を隠さない
- 本文、表、図、補足資料で数値を一致させる
- 有意差の有無だけでなく、効果量、信頼区間、不確実性を示す
- exploratory analyses は confirmatory analyses と区別する
- 結果の意味づけは Discussion で行い、Results では事実記述に留める

### 3.7 Discussion

Discussion は結果の言い換えではありません。出発点だった理論的問題に戻り、**結果がその問題に何をもたらしたのか**を示す場所です。

良い考察で行うこと:

- 研究課題への答えを、理論的な言葉で言い直す
- 主要結果をまとめて解釈する
- どの理論をどう修正、拡張、限定したかを示す
- 実践的含意を、データから言える範囲で述べる
- 限界と今後の課題を、形式的でなく実質的に示す

問いの型ごとの注意:

- 記述研究なら、実態の範囲を超えて因果を言わない
- 関連研究なら、関連と因果を混同しない
- 因果推定研究なら、必要な前提と限界を明示する
- 予測研究なら、説明より予測性能、校正、外部妥当性を中心に議論する
- シミュレーション研究なら、「この条件下での性能」であることを明示する
- メタアナリシスなら、統合できた理由、できなかった理由、異質性、バイアスの影響を議論する

Discussion で典型的に落ちる失敗:

- **Rehashing**
  - 結果の要約を繰り返すだけで、意味を解釈しない
- **Meandering**
  - 含意をたくさん挙げるが、互いにつながらず焦点がない
- **Overreaching**
  - データが支えていない大きな主張に飛びすぎる

## 4. 研究タイプ別の書き分け

### 4.1 観察研究

観察研究では、まず自分が何をしたいのかを分けることが重要です。

- 実態を記述したいのか
- 関連を見たいのか
- 因果効果を推定したいのか
- 予測したいのか

特に明示したい項目:

- ソース集団と解析集団の定義
- 時点0（time zero）
- 曝露定義とアウトカム定義
- フォローアップ期間と打ち切り規則
- どの変数を交絡候補として調整するか
- どの変数は調整しないか
- 欠測対応
- 感度分析

特に注意したいこと:

- 因果を言いたいのに、時間順序が曖昧
- 交絡因子と媒介因子を区別せずに調整する
- 主要曝露の定義を結果を見ながら変える
- 感度分析をうまくいったものだけ残す

### 4.2 既存データを使う研究

電子カルテ、レジストリ、請求データ、調査二次利用などでは、解析モデル以前に、**そのデータでそもそも何が観測でき、何が観測できないか**が主論点です。

特に必要な項目:

- データベースの性質と収集目的
- データ抽出日、版、更新時点
- 変数の取得可能期間
- 対象者抽出コード、アルゴリズム
- 曝露、アウトカム、共変量のコード一覧
- コードやアルゴリズムの妥当性根拠
- データ連結の方法、成功率、品質評価
- データクリーニングの手順
- データ欠落が欠測なのか、そもそも記録されないのかの区別
- 代表性と一般化可能性の限界

著者として特に気を付ける点:

- 使える変数があったから入れた、で済ませない
- コードリストやアルゴリズムは補足資料に逃がしてもよいが、存在は明示する
- データ提供元の制約で開示できない部分があれば、その旨と影響を書く
- 既存データにない概念を、あるように書かない

### 4.3 既存データを用いたシミュレーション研究

既存データを使ったシミュレーション研究では、少なくとも次の2パターンがあります。

- 実データからパラメータを推定し、それをもとに条件を設定する研究
- 実データの構造や分布を参考に、より一般化した人工データを発生させる研究

重要なのは、**実データから何を借りて、何を研究者が仮定したのかを分けて書くこと**です。

必須に近い項目:

- 研究目的
- ベースケースと追加シナリオ
- 実データから借りたパラメータと根拠
- 研究者が置いた仮定と理由
- データ生成機構の数式または手順
- 比較する方法の定義
- 性能指標
- 反復回数とその根拠
- Monte Carlo error への配慮
- 乱数シードや乱数状態の管理
- 収束失敗や数値エラーの扱い

ありがちな弱点:

- シナリオ設定の理由がない
- 反復回数の理由がない
- 収束失敗を報告しない
- 実データ由来の仮定と便宜的仮定が区別されていない
- 結果が良い条件だけ本文で強調される

### 4.4 系統的レビュー / メタアナリシス

メタアナリシスでは、analysis plan は統計モデル選びのメモでは足りません。**検索前から統合後までつながった計画**である必要があります。

最低限必要な項目:

- レビュークエスチョン
- 適格基準
- 情報源と検索戦略
- 選定手順
- データ抽出手順
- リスクオブバイアス評価法
- 効果指標
- データ変換ルール
- どの研究をどの synthesis に入れるかの基準
- 統合モデルの選択と根拠
- 異質性の評価法
- サブグループ解析、メタ回帰の計画
- 感度分析
- 出版バイアスや small-study effects への対応
- certainty の評価法
- どの条件なら統合しないか

特に気を付ける点:

- どの効果量を優先採用するか
- 複数時点、複数尺度、複数モデルからどれを採るか
- 同一研究の重複報告や重複集団をどう扱うか
- 依存した効果量をどう扱うか
- ゼロイベント研究や欠落した要約統計量をどう扱うか
- fixed-effect と random-effects をどう使い分けるか
- 異質性が強いとき、統合しない判断をどうするか

### 4.5 予測モデル研究

予測研究では、関連研究や因果研究の書き方をそのまま流用するとずれます。

明示したい項目:

- 開発研究か外部検証研究か
- データ分割法
- 情報漏洩の防止策
- 特徴量前処理
- ハイパーパラメータ調整法
- 性能指標
- 内的検証と外的検証の区別
- calibration と discrimination の両方
- intended use と decision context

よくある弱点:

- テストデータを何度も見て改善してしまう
- AUC だけで結論を書く
- calibration を見ない
- 内部検証だけで「有用」と言う
- 複数モデルを試して一番良いものだけを報告する

### 4.6 質的研究・混合研究法

質的研究では「SAP」という呼び名がしっくり来ないこともありますが、**後出しの解釈自由度を減らす**という発想自体は重要です。analysis protocol として、次を明示するとよいです。

- データ生成の場面と文脈
- サンプリング方針
- コーディング単位
- 分析手順
- 研究者間確認のやり方
- 反証例の扱い
- 混合研究法なら統合タイミングと統合方法

## 5. 先行研究と引用の使い方

### 5.1 引用の役割

引用は、参考文献欄を埋めるための装飾ではありません。少なくとも次の役割があります。

- 問題設定を正当化する
- 概念を定義する
- 既知の知見を整理する
- ギャップを作る
- 自分の貢献を位置づける
- 方法選択を正当化する
- 結果解釈を慎重にする

### 5.2 文献は「役割で選ぶ」

関連しそうな文献を多く入れるほどよいわけではありません。各引用に役割がある方が強いです。

よく使う組み合わせ:

- 古典研究 1本
- レビュー / メタ分析 1本
- 近年の代表研究 2〜4本
- 方法論文 1〜2本

広い主張には、個別研究を大量に並べるより、まずレビューやメタ分析を置いた方が読みやすく、説得力も高くなります。

### 5.3 段落の主役は引用ではなく、あなたの議論

段落の基本形:

1. 段落冒頭で、その段落の主張を言う
2. 必要な先行研究を統合して示す
3. 研究間の意味、限界、ズレを解釈する
4. 本研究の問い、仮説、含意に橋渡しする

この順番が崩れると、段落は文献メモになります。

### 5.4 よくある悪い書き方

- 一文一研究の連続
- 文献を大量投入して何を言いたいのかが消える
- 文章はあるが、実質的には記述的な羅列
- 同じ出典を毎文に機械的に反復する

原則として、引用は「一文ごと」ではなく「主張ごと」に考えます。どの情報、どの主張が、どの出典に依拠しているかが明確なら、毎文末に同じ引用を繰り返す必要はありません。

### 5.5 直接引用は最小限

心理学・社会科学の研究論文では、先行研究は通常、**直接引用より言い換えや要約**で扱います。直接引用が必要なのは、定義の wording 自体が論点である場合や、特定表現への批判や精査を行う場合などに限られます。

### 5.6 一つの段落に何本まで引用してよいか

厳密な上限はありません。ただし、実務的には次のように考えると整いやすいです。

- 狭い主張なら 1〜3本で十分なことが多い
- 広い合意ならレビューやメタ分析を中心に置く
- 理論的対立や境界条件を見せたい段落ではやや多めでもよい
- それでも、引用数より**統合の明確さ**を優先する

## 6. AI を使うときの実務

### 6.1 AI で起きやすい失敗

- すべての文に引用を付ける
- 一文一研究の連続になる
- 引用数が多いほどよいと誤解する
- 分析的でなく記述的になる
- 出典と主張の対応がずれる
- 存在しない文献を作る

### 6.2 AI に任せてもよいこと

- 文章の圧縮、言い換え
- 段落構成の提案
- 導入や考察の論理の並べ替え
- 既に確認済みの文献リストを使った草稿化

### 6.3 AI に任せない方がよいこと

- 文献の存在確認
- 引用内容の正確性判断
- 研究間の重要度の最終判断
- どの主張がどの文献に本当に支えられているかの確認

### 6.4 AI への指示例

以下のように、かなり具体的に指示した方が失敗しにくいです。

> 以下の確認済み文献リストだけを使って、導入の第2段落を書いてください。  
> 各段落は  
> 1) 段落の主張  
> 2) 先行研究の統合  
> 3) ギャップまたは本研究への橋渡し  
> の順で構成してください。  
> 一文一研究の列挙にしないでください。  
> 引用は必要最小限にし、同じ出典群が続く場合は毎文に繰り返さないでください。  
> 存在しない文献は絶対に作らないでください。  
> 著者名の羅列ではなく、研究間の共通点・相違点・限界を明示してください。

### 6.5 AI 使用後の最低チェック

- 各引用文献は実在するか
- 各主張はその文献に本当に支えられているか
- 段落冒頭に主張があるか
- 一文一研究の列挙になっていないか
- 最後に本研究への橋渡しがあるか
- 自分の解釈と先行研究の要約が混ざっていないか

### 6.6 ジャーナルごとの AI 開示

AI 利用の扱いはジャーナルごとに異なります。使ったなら、隠すよりも、

- どの範囲で使ったか
- どの範囲を人間が確認したか
- 引用や内容の最終責任を誰が負うか

を整理しておく方が安全です。

## 7. 査読対応に備えるための執筆ポイント

トップジャーナル向け原稿では、投稿後にうまく答えるより、**投稿前の段階で査読コメントの主要論点を潰しておく**方がはるかに重要です。

### 7.1 投稿前に準備しておくもの

- 研究課題、ギャップ、貢献、射程を一枚で示したメモ
- タイトル、要旨、導入、Methods、Results、Discussion の整合表
- 該当する報告ガイドラインのチェックリスト
- analysis plan / SAP / protocol / preregistration の所在
- 主要解析と感度分析の一覧
- 主要図表の見出しと本文中の参照対応
- データ辞書、コードリスト、抽出アルゴリズム、補足資料案
- 限界と、それでもなお言えることを切り分けたメモ

### 7.2 査読でよく問われる点と、先回りの仕方

- **この問いはなぜ重要か**
  - 導入の前半で広い意義を明示する
- **何が新しいのか**
  - 先行研究との差分を、データ、理論、設計、分析のいずれかで明示する
- **仮説や研究課題の論理が弱い**
  - 理論節で因果経路、境界条件、対立仮説を整理する
- **Methods が不透明**
  - 対象、定義、前処理、欠測、除外、モデル選択、ソフトウェアを明示する
- **観察研究なのに因果を言いすぎている**
  - 時間順序、交絡、仮定、限界を明示し、表現も調整する
- **探索的分析を確証的に見せている**
  - 事前指定と事後追加を区別する
- **existing data の定義が曖昧**
  - コードリスト、アルゴリズム、連結手順、データ欠落の性質を示す
- **シミュレーション研究のシナリオ設定が恣意的**
  - 実データ由来の前提と便宜的仮定を分ける
- **メタアナリシスの統合方針が後付けに見える**
  - 効果量選択、統合モデル、統合しない条件を先に書く
- **予測研究なのに calibration や外部妥当性が弱い**
  - discrimination と calibration を両方示し、内部検証と外部検証を区別する

### 7.3 Response letter の基本原則

査読返信では、反論よりも説明責任が重要です。

- コメントを一つずつ分けて答える
- 何を変えたか、何を変えていないかを区別する
- 変更したなら、本文のどこをどう変えたかを具体的に書く
- 変えていないなら、なぜそう判断したかを丁寧に説明する
- レビューアーの誤読を責めず、原稿側の説明不足として扱う
- できないことは無理にできると装わず、限界として明示する

### 7.4 「修正した」と「明確化した」を分ける

Response letter では、次の区別を明確にした方がよいです。

- **内容を変更した**
  - 解析を追加、表を追加、文言を修正、補足資料を追加
- **既存内容を明確化した**
  - もともと行っていたが、書き方が不十分だったため明示化した

この区別が曖昧だと、査読者は「本当に事前にやっていたのか」「都合よく後から足したのか」を疑いやすくなります。

### 7.5 返信文で避けたいこと

- 感情的な反論
- 相手の理解不足に責任を押し付ける表現
- 変更した箇所を特定できない曖昧な返答
- 「重要ではない」とだけ言って切る対応
- データが支えていない主張を、修辞で押し切ろうとすること

### 7.6 査読対応に強い原稿の特徴

- 主張が少数の重要点に絞られている
- Methods と Results の対応が明確
- 限界が形式的でなく、本当に解釈に関わる点を押さえている
- 補足資料に逃がすべき技術情報が整理されている
- どの設計判断が事前規定で、どれが追加対応かを説明できる

## 8. 実際に使えるテンプレート

### 8.1 導入段落

> [現象] は、[理論的 / 実践的理由] から重要です。既存研究は、主として [既知の点] を示してきました。しかし、[不足・矛盾・未解決点] は依然として十分に説明されていません。そこで本研究は、[方法や視点] を通じてこの問題に答えます。

### 8.2 理論節の段落

> 既存研究は、[段落の中心主張] を示唆しています。具体的には、[研究群の共通点・差異を要約]。ただし、[限界・境界条件・矛盾]。したがって、本研究では [仮説 / 研究課題] を検討します。

### 8.3 考察の段落

> 本研究の結果は、[研究課題への答え] を示しています。重要なのは、この結果が [理論名 / 概念] を [拡張 / 修正 / 限定] する点です。とくに、[複数結果をまとめた解釈]。このことは、[学術的 / 実践的含意] を持ちます。

### 8.4 査読返信の書き出し

> ご指摘ありがとうございます。ご懸念は、[論点] が原稿中で十分に明示されていなかった点にあると理解しました。これを受けて、[修正内容] を行い、[節 / 表 / 補足資料] に反映しました。

### 8.5 査読返信で変更しない場合の書き方

> ご指摘の重要性は認識しています。本研究では [設計上の理由 / データ上の制約] により [追加分析や変更] は行っていませんが、その限界が読者に明確になるよう、[節] に説明を追加しました。

## 9. 提出前セルフチェック

提出前に、少なくとも次の問いにすべて「はい」で答えられるか確認すると、原稿の質がかなり上がります。

- この論文の研究課題を一文で言えるか
- 導入の最後で貢献が明示されているか
- 理論節は文献レビューではなく論証になっているか
- 各段落の冒頭に、その段落の主張があるか
- 一文一研究の列挙段落が残っていないか
- 広い主張に対して、レビューやメタ分析を使えているか
- 方法は研究課題に適合しているか
- 結果は研究課題に答える順に並んでいるか
- 考察は結果の再要約で終わっていないか
- 含意は少数の重要点に絞られているか
- データが支えていない主張をしていないか
- 研究の型に応じた表現になっているか
- 欠測、除外、逸脱、前処理が説明されているか
- 確証的解析と探索的解析が区別されているか
- 透明性に関する必要情報が入っているか
- AI を使った場合、その扱いを説明できるか
- 該当する報告ガイドラインの必須項目が落ちていないか
- 査読で問われそうな主要論点に、本文か補足資料で答えられるか
- 参考文献と本文引用が一対一に対応しているか

---

## 10. まとめ

トップジャーナル論文を書くときに最も大事なのは、文献をたくさん知っているように見せることではありません。大事なのは、**重要な問いに対して、既存研究を正確かつ選択的に使いながら、自分の論理を前に出し、その論理に見合った方法と透明性で支えること**です。

その意味で、次の4点が中核になります。

- 研究の問いの型を最初に固定する
- 原稿と analysis plan を整合させる
- 文献レビューを記述的ではなく分析的に書く
- 査読で問われる論点を先回りして潰す

「引用をたくさん入れた文章」と「先行研究をうまく使った文章」は別物です。トップ誌で通るのは、後者です。
